현재 Unity 프로젝트를 분석해서 발톱 복원 ASMR 모바일 게임의 전체 아키텍처를 설계해줘.

중요:
- 먼저 현재 프로젝트의 Scene, Script, Prefab, Canvas, UI Hierarchy, 폴더 구조를 분석해.
- 바로 코드를 작성하거나 파일을 수정하지 마.
- 첫 응답에서는 분석 결과와 구현 계획만 보여줘.
- 내가 승인하기 전까지 Script, Scene, Prefab, ScriptableObject Asset을 생성하거나 수정하지 마.
- 설명, 계획, 질문은 한국어로 작성해.
- 코드, 클래스명, 변수명, 파일명, 커밋 메시지는 영어로 작성해.
- 코드 주석은 꼭 필요한 경우에만 영어로 작성해.
- 기존 코드를 최대한 재사용하고, 기존 오브젝트를 근거 없이 삭제하거나 이름을 변경하지 마.
- 불필요한 추상화, 인터페이스, DI 프레임워크, 이벤트 버스, 서비스 로케이터를 만들지 마.
- MonoBehaviour 싱글톤을 남발하지 마.
- 현재 프로젝트의 Unity 버전과 C# 버전에 맞는 문법을 사용해.
- target-typed new(), record, nullable reference type 등 지원 여부가 불확실한 최신 문법은 피하고 명시적인 문법을 사용해.
- 외부 라이브러리는 추가하지 마.
- Odin Inspector, UniTask, DOTween, Addressables는 사용하지 마.
- MVP 기준으로 단순하지만 확장 가능한 구조를 만들어.
- 기존 프로젝트에 동일하거나 비슷한 기능이 있으면 새로 만들지 말고 재사용 또는 최소 수정해.
- 작업 전 반드시 git status를 확인하고, 기존 사용자 변경사항을 덮어쓰지 마.

==================================================
1. 게임 핵심 방향
==================================================

게임 콘텐츠 비중:

- 발톱 복원 플레이: 60%
- 국가별 네일 디자인 수집: 25%
- 복원샵 및 살롱 성장: 15%

핵심 플레이 흐름:

1. 상태 진단
2. 오염 제거
3. 손상 부위 제거
4. 손상 발톱 복원
5. 발톱 모양 선택
6. 국가별 네일아트
7. 전후 비교
8. 디자인 도감 등록
9. 코인 및 살롱 성장 보상

단, 모든 스테이지에서 처음부터 전체 과정을 수행하지 않는다.

스테이지가 올라갈수록 이전 작업을 유지하면서 새로운 시술 단계와 난이도 규칙이 누적된다.

예시:

Stage 1
- Diagnosis

Stage 2
- Diagnosis
- DirtRemoval

Stage 3
- Diagnosis
- DirtRemoval
- DamagedNailRemoval

Stage 4
- Diagnosis
- DirtRemoval
- DamagedNailRemoval
- NailRepair

Stage 5
- 이전 과정
- NailShape 추가

Stage 6
- 이전 과정
- NailArt 추가

이후 스테이지:
- 같은 작업의 난이도 상승
- 복합 손상
- 숨겨진 오염
- 작은 작업 영역
- 움직이는 타깃
- 제한 시간
- 정확한 도구 순서
- 여러 고객 요청 조건
- 두 개 이상의 동시 작업 영역
- 희귀 고객 및 보스 고객

전후 비교, 도감 등록, 살롱 성장 보상은 TreatmentType 단계가 아니라 스테이지 완료 후 결과 처리로 구현한다.

==================================================
2. 현재 Scene 구조
==================================================

현재 Scene은 정확히 3개다.

1. Main Scene
- 시작하기 버튼이 있다.
- 시작하기 버튼을 누르면 Stage Scene으로 이동한다.

2. Stage Scene
- Stage Map 화면이 있다.
- Stage Map에서 숫자 스테이지를 선택한다.
- 스테이지 선택 후 같은 Stage Scene 안에서 Repair Shop Lobby Panel을 연다.
- 복원샵 로비에서 룰렛을 돌려 해당 스테이지의 고객을 선택한다.
- 룰렛 결과 고객이 확정되면 InGame Scene으로 이동한다.

3. InGame Scene
- 선택된 고객과 선택된 스테이지 설정을 바탕으로 발톱 복원 플레이를 진행한다.
- 완료 후 결과 화면을 표시한다.
- 결과 화면에서 Stage Scene의 복원샵 로비 또는 스테이지맵으로 돌아갈 수 있다.

중요:
- Main, Stage, InGame 3개 Scene 구조를 유지해.
- 새로운 Scene을 만들지 마.
- Stage Map과 Repair Shop Lobby는 Stage Scene 내부 Panel 전환으로 구현해.
- 룰렛 결과창도 Stage Scene 내부 Popup으로 구현해.
- 결과 화면은 InGame Scene 내부 Panel로 구현해.
- 현재 Scene 이름이 실제로 Main, Stage, InGame인지 프로젝트에서 확인하고, 다르면 실제 이름을 기준으로 설계해.

전체 흐름:

Main Scene
-> Start Button
-> Stage Scene
-> Stage Map Panel
-> Stage 선택
-> Repair Shop Lobby Panel
-> 고객 룰렛
-> 고객 결과 확정
-> InGame Scene
-> Treatment Gameplay
-> Result Panel
-> Stage Scene 복귀 또는 Main Scene 복귀

3. 데이터 설계 원칙
==================================================

ScriptableObject는 변경되지 않는 원본 게임 데이터에 사용한다.

PlayerSaveData는 플레이어의 영구 진행 상태에 사용한다.

GameSessionContext는 Scene 사이의 현재 선택 정보를 임시로 보관한다.

역할을 명확히 분리해.

ScriptableObject:
- 스테이지 정의
- 고객 정의
- 도구 정의
- 네일 디자인 정의
- 살롱 시설 정의
- 난이도 정의
- 보상 정의

SaveData:
- 코인
- 고객 클리어 여부
- 최고 별점
- 최고 점수
- 해금한 네일
- 살롱 레벨
- 시설 레벨
- 도구 레벨
- 직전 룰렛 고객
- 무료 회전 횟수
- 최초 보상 수령 여부

GameSessionContext:
- 현재 선택된 WorldData
- 현재 선택된 StageGroupData
- 현재 선택된 CustomerStageData
- 현재 선택된 NailDesignData
- InGame 완료 후 돌아갈 Stage Scene 화면 상태

ScriptableObject에 런타임 진행 상태를 저장하지 마.

SaveData에 ScriptableObject 참조를 저장하지 말고 ID 문자열만 저장해.

13. RuntimeTreatmentStepData
==================================================

CustomerStageData의 원본 TreatmentStepData에 난이도를 직접 덮어쓰지 마.

런타임 실행용 복사 데이터인 RuntimeTreatmentStepData를 일반 C# class로 구현해.

필드 예시:

- TreatmentStepData source
- TreatmentType treatmentType
- ToolData requiredTool
- float requiredProgress
- float timeLimit
- int maxMistakes
- int targetCount
- int simultaneousTargetCount
- float targetScale
- float toolEfficiencyMultiplier
- int requiredAccuracyPercent
- bool useHiddenTargets
- bool useMovingTargets
- bool requireCorrectToolOrder

BuildRuntimeSteps에서 생성한다.

생성 규칙:

1. StageGroupData.enabledTreatmentTypes 순서를 순회
2. CustomerStageData에서 같은 TreatmentType을 찾음
3. 찾은 단계만 RuntimeTreatmentStepData로 복사
4. StageDifficultyData 적용
5. 필수 단계가 없으면 명확한 경고 출력
6. 원본 ScriptableObject나 Serializable data는 수정하지 않음

19. GameSessionContext
==================================================

GameSessionContext를 구현해.

역할:
- Scene 간 현재 선택 데이터 전달
- 영구 저장 아님
- Stage Scene에서 선택한 데이터를 InGame Scene으로 전달
- InGame 완료 후 Stage Scene 복귀 위치 전달

보관 데이터:

- WorldData selectedWorld
- StageGroupData selectedStageGroup
- CustomerStageData selectedCustomer
- NailDesignData selectedNailDesign
- StageReturnTarget returnTarget
- bool reopenRepairShopLobby

필요 메서드:

- SetSelectedWorld(WorldData world)
- SetSelectedStageGroup(StageGroupData stageGroup)
- SetSelectedCustomer(CustomerStageData customer)
- SetSelectedNailDesign(NailDesignData design)
- PrepareStageReturn(StageReturnTarget target)
- ClearCustomerSelection()
- ClearAll()

중요:
- DontDestroyOnLoad를 사용해도 되지만 인스턴스는 하나만 존재해야 한다.
- 중복 인스턴스 방지
- 모든 Manager를 싱글톤으로 만들지 마.
- selectedCustomer 또는 selectedStageGroup이 없으면 InGame Scene 진입 금지
- 기존 프로젝트에 비슷한 Session Manager가 있으면 재사용 검토

23. 고객 룰렛
==================================================

복원샵 로비의 고객 선택은 일반 카드 선택이 아니라 룰렛 방식이다.

룰렛 처리 순서:

1. 유효 고객 후보 생성
2. 가중치 기반으로 결과 고객을 먼저 결정
3. 룰렛 애니메이션 시작
4. 감속
5. 이미 결정된 고객 위치에서 정지
6. 결과 고객 강조
7. RouletteResultPopup 표시
8. 사용자가 Start Treatment를 누르면 InGame Scene 이동

룰렛 물리 회전 결과를 기준으로 고객을 결정하지 마.

RouletteController 역할:

- Initialize(StageGroupData stageGroup)
- BuildCandidateList()
- SelectCustomerByWeight()
- ApplyCompletedCustomerPenalty()
- ApplyPreviousCustomerPenalty()
- Spin()
- ConfirmResult()
- ResetSpinState()

룰렛 규칙:

- 미완료 고객은 기본 가중치 사용
- 완료 고객은 가중치 감소
- 직전 고객은 가중치 감소
- 잠긴 고객은 후보 제외
- minimumClearCount 조건 미충족 고객 제외
- 유효 후보가 없으면 회전하지 않음
- 같은 고객 연속 등장 확률 완화
- 모든 고객 완료 후에는 전체 고객을 다시 정상적으로 선택 가능
- 연속 클릭 방지
- 회전 중 입력 잠금
- 테스트용 선택적 seed 지원
- UnityEngine.Random 또는 System.Random 중 하나만 일관되게 사용

RouletteView 역할:

- 고객 아이콘 배치
- 룰렛 회전 애니메이션
- 감속
- 지정 결과 위치에서 정지
- 결과 고객 강조
- 애니메이션 완료 이벤트 전달

현재 DOTween을 사용하지 않는 프로젝트라면 Coroutine으로 구현해.

RouletteResultPopup 표시:

- 고객 이름
- 고객 이미지
- 발 상태 미리보기
- 고객 요청 문구
- Start Treatment 버튼
- Respin 버튼
- Back 버튼

수익화는 현재 구현하지 마.

다음 확장 훅만 준비해:

- 무료 재회전
- 코인 재회전
- 보상형 광고 재회전

광고 SDK는 연결하지 마.
